# AnyLink 项目改进计划

> 本文档列出 AnyLink 项目中已确认的安全漏洞、代码缺陷和改进事项，按优先级分组排列，供后续逐个修复使用。  
> 每个条目包含：问题描述、涉及文件/行号、修复建议。

---

## 阶段一：关键安全漏洞（立即修复）

### 1.1 LDAP TLS 证书验证被禁用
- **严重级别**: 🔴 关键
- **文件**: `server/dbdata/userauth_ldap.go:94`
- **问题**: `InsecureSkipVerify: true` 完全禁用了 LDAP 连接的 TLS 证书校验，存在中间人攻击风险。
- **修复建议**:
  - 将 `InsecureSkipVerify` 改为可配置项，默认为 `false`
  - 在 `ServerConfig` 中新增 `LdapInsecureSkipVerify` 字段
  - 支持配置自定义 CA 证书路径

### 1.2 SMTP TLS 证书验证被禁用
- **严重级别**: 🔴 关键
- **文件**: `server/admin/common.go:89`
- **问题**: SMTP 发送邮件时同样使用了 `InsecureSkipVerify: true`。
- **修复建议**: 与 1.1 相同，改为可配置并默认关闭。

### 1.3 XML 模板 XSS 注入漏洞
- **严重级别**: 🔴 关键
- **文件**: `server/handler/link_auth.go:197-216`
- **问题**: `Group`、`Error`、`Groups` 字段未经 XML 转义直接渲染到 XML 响应中。当前仅 `Banner` 字段做了 `xml.EscapeText` 处理。
- **修复建议**:
  - 在 `tplRequest()` 函数中对 `Group`、`Error` 字段做 `xml.EscapeText` 处理
  - 对 `Groups` 列表中的每个元素也做转义
  - 考虑统一使用 `encoding/xml` 的结构体序列化来代替文本模板

### 1.4 请求体无大小限制（DoS 风险）
- **严重级别**: 🔴 关键
- **文件**: `server/handler/link_auth.go:40`、`server/admin/api_cert.go:72`
- **问题**: `io.ReadAll(r.Body)` 无大小限制，攻击者可发送超大请求导致内存耗尽。
- **修复建议**:
  - 替换为 `io.ReadAll(io.LimitReader(r.Body, maxSize))`
  - 认证请求建议限制为 1MB，证书上传建议限制为 10MB

### 1.5 TODO "严重问题" 未解决
- **严重级别**: 🔴 关键
- **文件**: `server/dbdata/user.go:100-101`
- **问题**: 被注释的 `return nil` 暗示可能存在绕过密码验证的逻辑缺陷。
- **修复建议**:
  - 仔细审查上下文，确认该 TODO 的原始意图
  - 如果 `return nil` 是错误遗留，确保已被正确移除
  - 删除 TODO 注释或添加说明注释

### 1.6 默认 JWT 密钥与管理员密码硬编码
- **严重级别**: 🔴 关键
- **文件**: `server/base/config.go:8-9`
- **问题**: `defaultJwt = "abcdef.0123456789.abcdef"` 和默认管理员密码哈希硬编码在源码中。JWT 密钥虽会自动重新生成，但管理员密码仅打印警告。
- **修复建议**:
  - 首次启动时强制要求设置管理员密码
  - 或在使用默认密码时自动生成随机密码并输出到日志
  - 从源码中移除硬编码的默认值

## 阶段二：高优先级缺陷（尽快修复）

### 2.1 CORS 配置过于宽松
- **严重级别**: 🟠 高
- **文件**: `server/admin/api_base.go:84`
- **问题**: `Access-Control-Allow-Origin: *` 允许任意来源跨域访问管理后台 API。
- **修复建议**:
  - 将允许的 Origin 改为可配置项
  - 默认仅允许同源访问
  - 生产环境禁止通配符

### 2.2 用户名枚举漏洞
- **严重级别**: 🟠 高
- **文件**: `server/dbdata/user.go:108-115`
- **问题**: 登录失败时返回不同的错误信息（"用户不存在"、"用户已过期"、"密码错误"），攻击者可通过差异化响应枚举有效账号。
- **修复建议**:
  - 统一返回通用错误信息，如 "用户名或密码错误"
  - 内部日志中记录详细原因以便调试

### 2.3 明文密码兼容模式
- **严重级别**: 🟠 高
- **文件**: `server/dbdata/user.go:134-141`
- **问题**: 通过密码长度（!= 60 字符）判断是否为明文密码，同时支持明文和 bcrypt 密码比对。
- **修复建议**:
  - 添加密码自动迁移逻辑：明文密码验证通过后自动转换为 bcrypt 存储
  - 在配置中添加 `AllowPlaintextPassword` 开关，默认关闭
  - 添加管理命令批量迁移明文密码

### 2.4 文件上传验证不足
- **严重级别**: 🟠 高
- **文件**: `server/admin/api_uploaduser.go:20-26`
- **问题**:
  - 文件名仅检查是否包含 `.xlsx`，未验证文件类型
  - 无 Content-Type 校验
  - 未检查 `header.Size`
  - `io.Copy` 错误被忽略
- **修复建议**:
  - 使用 `filepath.Ext()` 严格校验扩展名
  - 验证 `Content-Type` MIME 类型
  - 限制文件大小
  - 处理 `io.Copy` 错误

### 2.5 证书上传无大小限制
- **严重级别**: 🟠 高
- **文件**: `server/admin/api_cert.go:14-43`
- **问题**: 证书和密钥文件上传无 `ParseMultipartForm` 大小限制，无 MIME 校验，`io.Copy` 未限制大小。
- **修复建议**:
  - 调用 `ParseMultipartForm(maxSize)` 限制总大小
  - 使用 `io.LimitReader` 限制单文件大小
  - 校验证书/密钥文件的格式有效性

### 2.6 OTP 缓存无界增长
- **严重级别**: 🟠 高
- **文件**: `server/dbdata/user.go:165-185`
- **问题**: 全局 `userOtp map[string]time.Time` 无大小上限，高并发攻击下可耗尽内存。
- **修复建议**:
  - 添加最大条目数限制（如 10000）
  - 达到上限时拒绝新的 OTP 验证或清理最旧条目
  - 考虑使用 LRU 缓存替代原始 map

## 阶段三：中等优先级问题（计划修复）

### 3.1 strconv.Atoi 错误被忽略（19+ 处）
- **严重级别**: 🟡 中
- **涉及文件**:
  - `server/admin/api_ip_map.go:26-28`
  - `server/admin/api_set_audit.go:25-26`
  - `server/admin/api_group.go:26-28`
  - `server/admin/api_policy.go:26-28`
  - `server/admin/api_user.go:50-53`
  - `server/admin/api_uploaduser.go:74`
- **问题**: `id, _ := strconv.Atoi(idS)` 解析失败时默认为 0，可能返回错误数据。
- **修复建议**:
  - 检查错误并返回 400 Bad Request
  - 可封装一个通用的参数解析函数统一处理

### 3.2 模板执行错误被忽略
- **严重级别**: 🟡 中
- **文件**: `server/handler/link_auth.go:163-175`
- **问题**: `template.Parse()` 和 `t.Execute()` 的错误均被 `_` 忽略，失败时返回空响应。
- **修复建议**:
  - 将模板解析移到 `init()` 阶段（预编译），启动时即报错
  - 对 `Execute` 错误记录日志并返回 500 错误

### 3.3 CountAll 数据库错误被忽略
- **严重级别**: 🟡 中
- **文件**: `server/dbdata/db_orm.go:64-66`
- **问题**: `xdb.Count(data)` 错误被忽略，失败时返回 0。
- **修复建议**:
  - 修改 `CountAll` 返回 `(int, error)`
  - 调用方处理错误

### 3.4 配置路径转换被注释
- **严重级别**: 🟡 中
- **文件**: `server/base/cfg.go:114-124`
- **问题**: 绝对路径转换逻辑被注释掉，相对路径在不同工作目录下可能失效。
- **修复建议**:
  - 确定路径解析策略并取消注释，或
  - 添加文档说明必须在指定目录运行

### 3.5 Kubernetes 部署缺少健康检查
- **严重级别**: 🟡 中
- **文件**: `deploy/deployment.yaml`
- **问题**: 缺少 `livenessProbe` 和 `readinessProbe`，无法自动检测不健康的实例。
- **修复建议**:
  - 添加 HTTP 或 TCP 健康检查探针
  - 如需要，在服务端添加 `/healthz` 端点

### 3.6 证书管理 TODO 未完成
- **严重级别**: 🟡 中
- **文件**: `server/dbdata/cert.go:363, 380`
- **问题**: 默认证书和 SNI 回退逻辑标记为 TODO，功能可能不完整。
- **修复建议**:
  - 验证非 SNI 客户端的兼容性
  - 完善默认证书选择逻辑
  - 添加证书过期告警功能

### 3.7 IPv4 Only
- **严重级别**: 🟡 中
- **文件**: `server/pkg/arpdis/arp.go:4`
- **问题**: ARP 发现模块仅支持 IPv4。
- **修复建议**:
  - 评估 IPv6 支持的需求和优先级
  - 如需支持，实现 NDP（邻居发现协议）

## 阶段四：低优先级改进（持续优化）

### 4.1 ACL 检查空 if 块
- **严重级别**: 🟢 低
- **文件**: `server/handler/payload.go:75`
- **问题**: `if` 块为空，逻辑依赖 `else` 分支，代码可读性差。
- **修复建议**: 反转条件，消除空块。

### 4.2 环境变量映射未使用
- **严重级别**: 🟢 低
- **文件**: `server/base/config.go:96`
- **问题**: `envs` map 通过反射填充但使用率低，不利于 Docker 部署。
- **修复建议**:
  - 补全环境变量映射
  - 添加 Docker 环境变量配置文档

### 4.3 调试 TODO 注释清理
- **严重级别**: 🟢 低
- **涉及文件**:
  - `server/admin/api_base.go:17` — `// TODO 调试信息输出`
  - `server/handler/link_auth.go:24` — `// TODO 调试信息输出`
  - `server/handler/link_tunnel.go:37` — `// TODO 调试信息输出`
- **修复建议**: 评估是否仍需要，删除或替换为正式的日志级别控制。

### 4.4 测试覆盖率不足
- **严重级别**: 🟢 低（但长期重要）
- **当前状态**: 估计覆盖率 < 15%
- **缺失测试**:
  - Handler 层认证流程
  - Admin API 端点
  - LDAP/RADIUS 认证
  - 暴力破解保护
  - IP 分配和租约管理
- **修复建议**:
  - 逐步为关键路径添加单元测试
  - 优先覆盖安全相关的认证和鉴权模块
  - 添加集成测试覆盖完整的登录流程

### 4.5 缺少 API 版本控制
- **严重级别**: 🟢 低
- **问题**: API 路径无版本前缀（如 `/user/list` 而非 `/v1/user/list`），不利于后续迭代。
- **修复建议**: 考虑在下个大版本迭代时引入 API 版本路由。

### 4.6 文档完善
- **严重级别**: 🟢 低
- **缺失文档**:
  - API 接口参考文档
  - 全部配置项说明
  - 数据库表结构文档
  - 安全加固指南
  - 故障排查手册
  - 备份恢复流程
  - 版本升级/迁移指南
